Originally developed by Alfred Aho, Brian Kernighan, and Peter Weinberger in 1977, AWK is a pattern-matching language for writing short programs to perform common data-manipulation tasks. In 1985, a new version of the language was developed, incorporating additional features such as multiple input files, dynamic regular expressions, and user-defined functions. This new version is available for both Unix and MS-DOS. This is the first book on AWK. It begins with a tutorial that shows how easy AWK is to use. The tutorial is followed by a comprehensive manual for the new version of AWK. Subsequent chapters illustrate the language by a range of useful applications, such as: *Retrieving, transforming, reducing, and validating data *Managing small, personal databases *Text processing *Little languages *Experimenting with algorithms The examples illustrates the book's three themes: showing how to use AWK well, demonstrating AWK's versatility, and explaining how common computing operations are done. In addition, the book contains two appendixes: summary of the language, and answers to selected exercises. 020107981XB04062001

    system函数。在awk的action语句中可以用管道和重定向符的。
    用到管道的时候，可以直接在管道前后用双引号包裹的命令。但是管道符号前后不能都是外部命令。
    如print |“sort”
    或是“who“|getline
    关于这种方式，无意间还发现了一个很的现象：
    ____________________________
    1 {while("who"|getline>0)
    2 print $0}
    3 {while("who"|getline>0)
    4 print $0}
    5 {while("date"|getline>0)
    6 print $0}
    ____________________________
    执行以上内容的脚本时，发现who命令和date命令的内容都只会输出一次。无论awk作用的文件有多少行（也就是以上脚本会被执行多少次）。应该是awk中只执行了getline中的前置命令一次，保存在一个临时文件中，然后每次getline都从这个临时文件中读取。
    Answer：
    在awk中要多次使用一个管道时，注意根据程序需要关闭这个管道对应的由外部命令创建的临时文件。如上加入close("date"）和close("who")以后就可以达到每行都输出的目的。
    如果不关掉会有以下效果：
    如果该临时文件为输入文件，则该文件只会在多行record中只用创建一次（本例中的情况）；
    如果临时文件为输出文件，比如print $0|"cat"。“cat”这个pipe临时文件会读取每一行的输入。但是知道整个awk程序结束时，“cat"被强制关闭时，才会输出内容。如果要每行都输出。那就得每次都close。
    所以，在使用pipe或是其他文件的时候，都要注意及时关掉文件。
    除了管道，awk中执行外部命令其他情况都要用system函数了。
    当你要用到记录Record中的内容时，可直接用$i与字符串连接。
    比如
    cat $0;无效
    system("cat "$0)有效

    #第一章 AWK概述[chapter_1]
    awk表达能力很强，能广泛应用于各式计算和数据处理任务。本章仅仅是一个引子，旨在帮助你尽快投入awk实践。第二章会对awk的语法进行详尽的讲解。余下的章节会向你展现如何用awk解决不同领域的问题。纵贯本书，我们尽量挑选一些能让你感到有用、有趣、有益的示例。
    ## 第一节　我们开始吧[section_1]
    实际中，awk程序都很短，就一两行。后面我们会用到一个`emp.data`的数据文件，里面记录有雇员名字、时薪、工作时长，每个雇员记录占一行（每行数据称为一个“记录”），比如：
    Beth 4.00 0
    Dan 3.75 0
    Kathy 4.00 10
    Mark 5.00 20
    Mary 5.50 22
    Susie 4.25 18
    如果你想调出那些干了活的雇员名字和应付的费用（按小时计），找awk，这是它的拿手好戏。只需进入命令行敲入：
    awk '$3 > 0 { print $1, $2 * $3 }' emp.data
    就会得到如下输出：
    Kathy 40
    Mark 100
    Mary 121
    Susie 76.5
    这行命令是让系统启动awk，单引号之间的代码是所要运行的程序，所需处理的数据来自`emp.data`。现在我们来分析单引号之间的awk程序。这是一个**模式-动作**语句。模式即为`$3 > 0`，它将找出所有第三列（列，也称之为**域**）值大于零的记录，而后执行下列动作：
    { print $1, $2 * $3}
    这句的意思是打印该行的第一个域和第二、三域之积。
    如果想打印没有干活的雇员名字，敲入命令：
    awk '$3 == 0 { print $1 }' emp.data
    解释一下模式`$3 == 0`，若某条记录的第三域为零，即执行其后的动作——打印该记录的第一域：
    { print $1 }
    在你读我们这本书的同时，不要忘了亲自动动手，试试执行书中的例子，稍稍做些修改，你会收获更多。这本书里的大部分程序都很短，这样便于你更好地理解awk的运作方式。在Unix下，如果你执行上面两个示例，可能会出现这样的交互：
    $ awk '$3 > 0 { print $1, $2 * $3 }' emp.data
    Kathy 40
    Mark 100
    Mary 121
    Susie 76.5
    $ awk '$3 == 0 { print $1 }' emp.data
    Beth
    Dan
    $
    命令前的`$`是系统提示符，各个环境下可能不同。
    ### awk程序的结构[section_1_1]
    我们再来看看上面这两条命令，单引号之间的是awk程序。在这一章里面，我们姑且把awk程序看作一连串模式-动作语句：
    模式 { 动作 }
    模式 { 动作 }
    ⋯⋯
    awk就这样一行一行地扫描输入的数据，搜索匹配模式的数据记录。“匹配”的具体含义依问题而定。对于模式`$3 > 0`，匹配上的意思就是“比较判断为真”。
    每处理一条记录，awk就会逐一比对所有的模式。一旦匹配上就会启动相应的动作（可以很复杂）。然后再读取下一行，循环往复，直至处理完所有输入。
    上例就很好地反映了这一特点：比对模式-执行动作。
    $3 == 0 { print $1 }
    是一个模式-动作单句。凡第三域为零，则打印第一域。
    模式和动作这两部分并不都是必需的。可以只有模式，没有动作：
    $3 == 0
    这一句会打印所有符合模式（即第三域为零）的记录。用来处理`emp.data`就会得到如下输出：
    Beth 4.00 0
    Dan 3.75 0
    也可以只有动作，没有模式：
    { print $1 }
    这样就会打印所有记录的第一个域。
    正因为模式和动作都是可选的，所以必须用大括号将动作语句括住以以示区分。
    ## 如何执行awk程序[section_1_2]
    执行awk程序有好几种方式。一般是这样：
    awk '程序代码' 数据文件
    可以指定多个数据文件。如下：
    awk '$3 == 0 { print $1 }' file1 file2
    执行这句会对`file1`和`file2`的所有记录进行甄别，挑出所有第三域为零的记录，打印其第一个域。
    你也可以在终端里不填数据文件，
    awk '程序代码'
    这样awk就会把你之后敲入的字符当做数据，直至你敲入文件结束符(end-of-file signal)（Unix下是control-d）。下面是Unix下的一段交互：
    $ awk '$3 == 0 { print $1}'
    Beth 4.00 0
    Beth
    Dan 3.75 0
    Dan
    Kathy 3.75 10
    Kathy 3.75 0
    Kathy
    …
    第三、五、八行为终端输出的内容。
    你可以利用这个特性来好好观察awk是怎么运行的。你应该试试执行上面的示例，并做一些小的改动（看看是否与设想的不同）。
    可能你已经注意到了，程序都用了单引号括住。如果代码短还好说，一旦代码很长怎么办？这时，你可以把awk代码单独写到一个文本里面，假设它叫`progfile`，然后这样来执行：
    awk -f progfile 数据文件（可选）
    选项`-f`让awk抓取(fetch)一个叫`profile`的文件，执行代码在里面。`progfile`可以替换成任何文件名。
    ## 诊断提示[section_1_3]
    如果你的awk程序有错，awk会给出诊断提示。比如，你把左大括号敲成了左方括号：
    awk '$3 == 0 [ print $1 }' emp.data
    awk就会给出这样的提示：
    awk: syntax error at source line 1
    context is
    $3 == 0 >>> [ <<<
    extra }
    missing ]
    awk: bailing out at source line 1
    “syntax error”的意思是你犯了一个语法错误。awk还帮你把错误锁定在>>>和<<<之间。“bailing out”是说并没有试图恢复。awk有时会给出更多的出错信息，比如括号不匹配等等。
    如果遇到了语法错误，awk不会去执行这个程序。有些错误只有等程序执行了才能发现。比如，你试图去除零，这时awk会中止程序，然后报告这行代码试图除零，并会给出出错的行号。
