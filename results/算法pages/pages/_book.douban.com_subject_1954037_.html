内容简介 ······

《计算机体系结构量化研究方法》(英文版·第4版)系统地介绍了计算机系统的设计基础、指令集系统结构、流水线和指令集并行技术、层次化存储系统与存储设备、互连网络以及多处理器系统等重要内容。在这个最新版中，作者更新了单核处理器到多核处理器的历史发展过程的相关内容，同时依然使用他们广受好评的“量化研究方法”进行计算设计，并展示了多种可以实现并行性的技术，而这些技术可以看成是展现多处理器体系结构威力的关键!在介绍多处理器时，作者不但讲解了处理器的性能，还介绍了有关的设计要素，包括能力、可靠性、可用性和可信性。

    关于TLB，与L1/L2 cache作用的理解，在附录C和ch5中都有说明，首先来看附录C里的一张图。

    图C2.24
    Operating Systems design and implementation 的 chapter 4 memory management中没有讨论L1/L2 cache与TLB的相互的作用，只详细的介绍了TLB的工作原理和过程。而在《计算机系统结构--量化研究方法》 ed4的ch5和附录C中给出了详细的介绍。
    将这两部书相互参考阅读，就可以比较清晰的了解TLB与cache的工作原理了。不过尚有一点问题需要求证。
    现代操作系统运行与现代体系结构上的，虚拟内存机制(Virtual memroy)机制也是现代操作系统的基础。而VM机制与分页（paging)机制息息相关。（参考文献1 ch4）
    来具体讲讲cache和TLB的问题。
    程序运行时，运行于虚拟地址空间(Virtual Address Space)，每一条指令里涉及的地址都是虚拟地址（Virtual Address），而程序（指令和数据）是放在物理存储器（Physical Memory）上的，因此需要将虚拟地址转换成物理地址。
    “L1/L2cache中每一个块地址是物理地址。”
    指令中涉及到的地址都是虚拟地址，这就需要将虚拟地址转换成物理地址，为了加速这种转换过程，先从TLB里找根据虚拟地址所在的页（page）索引号对应的物理页框（page frame）索引号，如果TLB 命中，即此时已经得到了page frame index，再和虚拟地址中的offset拼接即得到物理地址，然后在L1/L2 cache中寻找。之后的过程大家耳熟能详，不再赘述。
    如果很不幸，发生了TLB缺失，那么CPU不得不从主存上的页表（通常有两级页表）中页表项中的信息得到物理地址，这会有两种情况：
    1）如果在页表项中，发现页缺失，这会触发page fault。在page fault handler中从物理磁盘调入页到主存，并更新页表中的页表项，TLB中的项也要更新，之后L2 cache与L1 cache也要依次更新。
    这种情况下，需要更新的地方，以及更新采用的算法分别为：
    主存中页表，（更新的是页表项，包括page index 与 page frame index，以及一些标志位）
    主存中可能需要将一些很久不用的页换出，然后将发生了缺失的页换入
    上述两个过程采用的是老化算法（aging），比较接近LRU算法的效果。
    CPU中MMU中TLB中的项（虚拟页索引与页框索引）（LRU算法）
    L2 cache（cache中的块，包括指令和数据） （LRU算法）
    L1 cache（cache中的块，包括指令和数据）（LRU算法）
    2）如果在页表项中信息表明页是有效的，也就是说此时没有发生页缺失。从页表项里得到物理页框索引号。在和虚拟地址中的偏移量组合记得到实际的物理地址。那么问题来了，此时CPU是如何操作呢？
    是回去从L1 cache和L2 cache中寻找，还是直接略过cache，先更新TLB中项，然后更新L2 cache，随后再更新L1 cache呢？ （见图C2.24)
    这个问题没想明白啊。各位不吝赐教。
    参考文献：
    [1]. Operating Systems design and implementation，ed3
    [2]. 《计算机系统结构--量化研究方法》 ed4, 白跃彬译

    关于TLB，与L1/L2 cache作用的理解，在附录C和ch5中都有说明，首先来看附录C里的一张图。

    图C2.24
    Operating Systems design and implementation 的 chapter 4 memory management中没有讨论L1/L2 cache与TLB的相互的作用，只详细的介绍了TLB的工作原理和过程。而在《计算机系统结构--量化研究方法》 ed4的ch5和附录C中给出了详细的介绍。
    将这两部书相互参考阅读，就可以比较清晰的了解TLB与cache的工作原理了。不过尚有一点问题需要求证。
    现代操作系统运行与现代体系结构上的，虚拟内存机制(Virtual memroy)机制也是现代操作系统的基础。而VM机制与分页（paging)机制息息相关。（参考文献1 ch4）
    来具体讲讲cache和TLB的问题。
    程序运行时，运行于虚拟地址空间(Virtual Address Space)，每一条指令里涉及的地址都是虚拟地址（Virtual Address），而程序（指令和数据）是放在物理存储器（Physical Memory）上的，因此需要将虚拟地址转换成物理地址。
    “L1/L2cache中每一个块地址是物理地址。”
    指令中涉及到的地址都是虚拟地址，这就需要将虚拟地址转换成物理地址，为了加速这种转换过程，先从TLB里找根据虚拟地址所在的页（page）索引号对应的物理页框（page frame）索引号，如果TLB 命中，即此时已经得到了page frame index，再和虚拟地址中的offset拼接即得到物理地址，然后在L1/L2 cache中寻找。之后的过程大家耳熟能详，不再赘述。
    如果很不幸，发生了TLB缺失，那么CPU不得不从主存上的页表（通常有两级页表）中页表项中的信息得到物理地址，这会有两种情况：
    1）如果在页表项中，发现页缺失，这会触发page fault。在page fault handler中从物理磁盘调入页到主存，并更新页表中的页表项，TLB中的项也要更新，之后L2 cache与L1 cache也要依次更新。
    这种情况下，需要更新的地方，以及更新采用的算法分别为：
    主存中页表，（更新的是页表项，包括page index 与 page frame index，以及一些标志位）
    主存中可能需要将一些很久不用的页换出，然后将发生了缺失的页换入
    上述两个过程采用的是老化算法（aging），比较接近LRU算法的效果。
    CPU中MMU中TLB中的项（虚拟页索引与页框索引）（LRU算法）
    L2 cache（cache中的块，包括指令和数据） （LRU算法）
    L1 cache（cache中的块，包括指令和数据）（LRU算法）
    2）如果在页表项中信息表明页是有效的，也就是说此时没有发生页缺失。从页表项里得到物理页框索引号。在和虚拟地址中的偏移量组合记得到实际的物理地址。那么问题来了，此时CPU是如何操作呢？
