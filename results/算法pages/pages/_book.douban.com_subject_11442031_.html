现在是 IT的时代，工作与生活都离不开IT。许多人都想了解软件原理，甚至编写一些简单的软件。通过《编程 ING：人人都能学会程序设计》一书，你就能拥有这样的能力，甚至培养出对编程的兴趣。

本书在编写上采用了心理学的一些技巧，每一页都有绘制精美的彩图并配有相应的说明文字，通过图的方式加强说明，而且图片与下面的文字相互呼应，达到重复的效果，可以加深记忆。读者在看图的过程中思考，在阅读文字的过程中加强理解，在动手实践的过程中实际体会，学习效果相当好。

这本书分成三大篇：第一篇讲述编程原理，第二篇讲述语法细节，第三篇是范例演示。第一篇重点在打破隔阂，让读者对编程有基本的认识，不再感到陌生或恐惧。第二篇详细说明语法，内容安排得深入浅出，不像一般程序设计书籍那般枯燥，而且强调动手做实验。第三篇的范例将前两篇的知识整合起来，并且突破性地设计渐进式的范例，让学习的坡度相当和缓，且讲解了所有的程序要素，包括循环、分支、函数、递归与模块化。

任何人想学程序设计，这本书都会是一个明智的选择。

    第6章 语境和单字

    LIB语境

    语境（context）是单字定义表（单字和定义的对应），单字的定义可能是程序（函数）或者一般的值。有些单字不影响其他值（也就是不需要参数），有些单字影响前面或后面的值（也就是需要参数）。 ？ LIB查看内部单字定义表

    USER语境

    LIB语境：提供常用的单字定义，用于写代码 USER语境：记录用户代码执行过程中发生的变化，我们所做的一切只会影响USER语境。? self可以查看USER语境 代码块：[]

    第7章 多语境的操作

    绑定新语境

    通过路径（path）的写法，指明某单字的语境来自哪里。例如syetem/option/home表示REBOL主目录。

    对象

    对象（object）用来将相关数据放在一起。写法如下： customer1: object [ name: “Atoms” birthday: 1990-1-1 ] REBOL中不区分对象和语境，作用完全一样。一般语境内放的单字较多，函数占大多数，值不固定，变化很多；对象内放的单字较少，数据占大多数，值则相对固定，就像数据库中的字段一样。 REBOL是Relative Expression Based Object Language，意思是此语言能够根据上下文（语境）做出不同的表达。

    第2篇 语法定义

    第8章：一切都是值

    三条规则之一：一切都是值。 把代码切割成许多值。值与值之间，用空白隔开，空白包括：空格，回车符，制表符，这三个字符都是不可显示字符。有些空白是多余的，可以省略，但多余的空格可以帮助阅读和理解（排版）。 代码排版的目标：方便阅读理解。详细的原则有三个：

    • 段落对齐
    • 意义对齐
    • 缩进表示包含或接续（缩进表示属于上面的段落，通常使用一个制表符或四个空格） 任何原则都不应该违背“让代码清晰易懂”的目的。

    第9章 数据类型

    第二条原则：每个值都属于某种类型。 在内存中，每个值都占用12个字节，外加4个字节的标签来记录值的类型。 数据类型（datatype）简称为类型（type），类型名称以！结尾。REBOL支持56种类型，可以分为间接值和直接值。 type? 查询类型。

    第10章 字面值

    第6章 语境和单字

    LIB语境

    语境（context）是单字定义表（单字和定义的对应），单字的定义可能是程序（函数）或者一般的值。有些单字不影响其他值（也就是不需要参数），有些单字影响前面或后面的值（也就是需要参数）。 ？ LIB查看内部单字定义表

    USER语境

    LIB语境：提供常用的单字定义，用于写代码 USER语境：记录用户代码执行过程中发生的变化，我们所做的一切只会影响USER语境。? self可以查看USER语境 代码块：[]

    第7章 多语境的操作

    绑定新语境

    通过路径（path）的写法，指明某单字的语境来自哪里。例如syetem/option/home表示REBOL主目录。

    对象

    对象（object）用来将相关数据放在一起。写法如下： customer1: object [ name: “Atoms” birthday: 1990-1-1 ] REBOL中不区分对象和语境，作用完全一样。一般语境内放的单字较多，函数占大多数，值不固定，变化很多；对象内放的单字较少，数据占大多数，值则相对固定，就像数据库中的字段一样。 REBOL是Relative Expression Based Object Language，意思是此语言能够根据上下文（语境）做出不同的表达。

    第8章：一切都是值

    三条规则之一：一切都是值。 把代码切割成许多值。值与值之间，用空白隔开，空白包括：空格，回车符，制表符，这三个字符都是不可显示字符。有些空白是多余的，可以省略，但多余的空格可以帮助阅读和理解（排版）。 代码排版的目标：方便阅读理解。详细的原则有三个：

    • 段落对齐
    • 意义对齐
    • 缩进表示包含或接续（缩进表示属于上面的段落，通常使用一个制表符或四个空格） 任何原则都不应该违背“让代码清晰易懂”的目的。

    第二条原则：每个值都属于某种类型。 在内存中，每个值都占用12个字节，外加4个字节的标签来记录值的类型。 数据类型（datatype）简称为类型（type），类型名称以！结尾。REBOL支持56种类型，可以分为间接值和直接值。 type? 查询类型。

    第10章 字面值

    具有字面值表达法的类型有30种，称为直接值。 integer! 可用做视觉分隔，如每三个数字就用一个隔开。 decimal! 可用,或.表示小数点，可以用E表示科学计数法，E表示10的n次方，最大小数点后16位。 percent! 以%结尾的数字。 money! 可以表达的范围比整数大，可以表达的精确度比小数高，达到10进制的26位。 pair! 两个数字中间放一个x，就成了数对，一般表示平面上点的坐标、平面的大小（宽和高）、屏幕分辨率。 tuple! 3~10个数，每个数介于0~255（刚好占用一个字节），用.连接，就成了数群，常表示颜色值或网络IP。例如，颜色用四个值表示，分别是红（R）、绿（G）、蓝（B）、透明（A）的比例。 char! 字符，内容在#””的引号中间（与第4章字符编码相关） string! 字符串有字符组成，除了’’ ‘’，还可以用{ }表示，而且可以串行，双引号不能串行 转义串：^后紧接着一个可显示字符，会被转成一个不可显示字符，如回车符、制表符，分别可以用^/和^-表示。 file! 文件（目录属于文件的一种），名称前有%。文件和目录的分隔符一律使用/，而不是用\。 如果文件是目录，一定以/结尾，如%my-dir/，否则不是目录。%后紧接着/表示绝对路径（如%/c/users），否则是相对路径（%REBOL/）。 文件中如果有空格等特殊字符，要用%加上其ASCII码（两个十六进制数字）代替，或者整体当成字符串，双引号最前面再加上%。表示磁盘时的冒号要去掉。如C:\Windows\Program Files，应该表示为%/c/Windows/Program%20Files/，或者%”/C/Windows/Program Files/“ url! 只要中间出现:，就会被视为网址。写法与目录类似，但是以网络协议开头，后面跟着冒号。若出现特殊字符，同样用%加其ASCII码（两个十六进制数字）代替。 email! 出现@，就是电邮。 tag! 用<>括起来，就是标签，前后可以有/，中间可以有空格分隔出属性。 time! 时间用冒号分隔，秒可以有小数。 date! 日期的分隔符号是-或/，按日月年或年月日的顺序，可以加上时区（前面要加上+或-），中间用/隔开。 binary! 二元是用来表达二进制格式的数据，通常是#{ }中间加上偶数个十六进制数字，也可以用二进制或64进制，同时前面加上2或64。二元的存在通常只是一个过程，后续可能通过解码的算法，转成图片、音乐、文字等内部格式（to-binary文字编码，to-string文字解码）。 block! 方块用[ ]括起来，中间可以放任何数据。方块会不会被执行，由方块前面的函数决定。方块中的数据时钝化的，不被主动计算。quote函数可以也保证数值不被计算。 paren! 方块用( )表示，是parenthesis的缩写，方块内的数据时活化的，会主动计算。园块最后一定有一个返回值。

    refinement! 单字前面加上/就成了修饰字，一般用在函数上，改变函数的行为。单字的首字符不可以是数字，但修饰字的首字符可以是数字。 issue! 修饰字的/ 换成#。首字符可以是数字。 set-word! 设字是单字后加: ，设字后的下一个值会被设定到单字，设字会再将结果输出。 get-word! 取字是单字前加 :符号，取字会取出单字的定义。如果定义是函数，则取出函数本身（而非函数计算后的结果，因此是实时计算的）。 lit-word! 单字前加 ‘符号（单引号），原字的计算结果是单字。

    path! 路径（与文件路径无关！）内可以包含多个值，这些值之间用/分隔。路径的第一部分一定是单字，后续部分可以是单字，数字，数对，设字，圆块。

    none!（空值），logic!（逻辑），unset!（未设）都： 1.字面值的写法都是#[ ]。 2.打印是都不打印字面值，而是打印好看的值 空值一般写成none，表示可以有值，但现在没值。 逻辑值一般写成true或false。 未设表示根本就不应该有值。

    第11章 间接值

    无法直接用字面表达在代码中，只能是计算过程中的产物（即函数的返回值）。间接值本身是一个内存地址，通过此地址找到真正的数据。

    第14章 函数计算

    函数是一段可执行的代码，执行的结果可能每次都不同。有的函数需要参数，有的函数不需要参数。函数一定有一个返回值，没有返回值时相当于返回特殊值unset。 REBOL没有先乘除后加减的规则，如op!（运算符）一律左侧先算。但通过括号可以改变顺序。op!的优先级高于正常函数与设字。 有的函数有修饰字可选用，修饰字会改变函数的行为。

    第17章 分支与循环

    bug是程序出现的问题，程序目前运行正常不代表没有bug，有可能在运行一段时间后在特定时间才会出问题。

    if是一个函数，它需要两个参数，一是条件，二是代码方块。if有一个修饰字/else，此修饰字需要一个参数，是条件不成立时要执行的代码方块。但实际使用either，因为路径需要动态绑定，效率比较差。

    函数调用语境。

    while函数：两个参数都是方块。第一个参数是条件，方块内的运行结果如果为真，则运行第二个代码。运行完再回到判断条件，如此反复，直至条件不成立。

    其他循环函数：for, loop, repeat, until, foreach, forall, forever, forskip。

    代码中最基本的三种执行顺序：顺序、分支、循环。一般的代码是顺序执行，if是条件执行，while是循环执行。
