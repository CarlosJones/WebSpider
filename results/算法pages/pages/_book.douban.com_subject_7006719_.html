搜索引擎作为互联网发展中至关重要的一种应用，已经成为互联网各个领域的制高点，其重要性不言而喻。搜索引擎领域也是互联网应用中不多见的以核心技术作为其命脉的领域，搜索引擎各个子系统是如何设计的？这成为广大技术人员和搜索引擎优化人员密切关注的内容。

本书的最大特点是内容新颖全面而又通俗易懂。对于实际搜索引擎所涉及的各种核心技术都有全面细致的介绍，除了作为搜索系统核心的网络爬虫、索引系统、排序系统、链接分析及用户分析外，还包括网页反作弊、缓存管理、网页去重技术等实际搜索引擎必须关注的技术，同时用相当大的篇幅讲解了云计算与云存储的核心技术原理。另外，本书也密切关注搜索引擎发展的前沿技术：Google的咖啡因系统及Megastore等云计算新技术、百度的暗网抓取技术阿拉丁计划、内容农场作弊、机器学习排序等。诸多新技术在相关章节都有详细讲解，同时对于社会化搜索、实时搜索及情境搜索等搜索引擎的未来发展方向做了技术展望。为了增进读者的理解，全书大量引入形象的图片来讲解算法原理，相信读者会发现原来搜索引擎的核心技术理解起来比原先想象的要简单得多。

作者简介 ······

张俊林：本科毕业于天津大学管理学院，2004年于中科院软件所直接获得博士学位并留所从事科研工作，研究方向为搜索引擎与自然语言处理。2005年在CSDN博客发布系列博文“搜索引擎设计实用教程：以百度为例”，在网络上获得了广泛转载与良好口碑。2006年作为联合创始人建立了智能信息聚合网站“玩聚网”，曾先后于阿里巴巴搜索技术中心任资深搜索技术研究员、房价网首席研究员，现任职于新浪微博，从事微博搜索与语义分析及推荐方面的研发工作。

    索引压缩
    词典压缩：原本单词为索引一列，列中每个单词的空间都分配最长单词的空间。现在不存单词，而是指向单词的地址。然后开一片连续空间存储所有单词，被指向。进一步优化，因为索引单词排序，单词词典排序，所以可以让相邻单词指针共享，但需要加分隔
    倒排列表压缩
    特点：存文档编号、单词位置差值，表中小整数多
    指标：压缩率，压缩速度，解压速度（最重要）
    Unary code: 对于整数 x，用 x-1 个 1 加一个表示结束的 0，共 x 个二进制数
    Binary code: n 位 bits 表示 2^n 内的数
    Elias Gamma: x = 2^e + d。 把 e+1 用 unary code，d 用 e 位 binary code
    Elias Delta: 继续把 e 也用 Elias Gamma 压缩
    Golomb & Rice: f1 = (x-1) / b, f2 = (x-1) mod b。然后对 f1+1 用 unary code，f2 用 binary code。Golomb 和 Rice 用不同的策略选择 b。局部 b 和全局 b。尚不明白为什么 x-1，继续压缩？范围从 0 开始？
    Variable byte: 128 进制数，1 byte 为一个单位，后 7 位为有效位，最高位标记是否为最后一个 byte
    SimpleX: 32 bits 压缩多个数，其中前 4 位指示后 28 位被均分成 X 份，每份用来压缩 0 ~ 2^x-1 之间的数。一次解压多个数
    PForDelta（最快）: 连续K（一般 128）个数一组，前 10% 大的为异常数。结构为：异常链表头，常规数据区，异常数据区。异常链表头记录到第一个异常数的间隔，原来的异常数们记录到下一个的间隔，最后一个异常数指向尾部的异常数据区，异常数据区里为原来的逆序
    DocID recording
    动机：文档编号小压缩效率高。所以存 D-Gap，重新编号希望倒排里表中相邻文档编号接近
    入手：相似网页包含重叠单词，希望把相似网页依次编号
    方法：文本聚类
    Static index pruning
    动机：有损压缩。搜索引擎只返回相关性最高的，所以希望丢弃不重要的部分，已达到更高的压缩效率。
    以单词为中心：在建好的倒排列表中，计算文档与单词的相关性，保留至少 K 个，去掉部分比例。效果不佳，因为相关性得分很可能相差不大，所以可以把所有的得分都减去最低的那一个分数，效果会有很大提升
    以文档为中心：将文档缩水，只保留重要单词，在此基础简历倒排列表。但是，一些单词的倒排列表会为空，比如停用词“的”
    检索模型与搜索排序
    Boolean Model
    查询词与文档的相关性是二元的，结果粗糙
    Vector Space Model
    经验型模型，缺乏明确理论来指导改进方向
    把文档看作 t 维特征向量，最常用单词作为特征，求其权重。实际常有成千上万维
    相似性计算
    cosine similarity: 特征向量空间中两个向量的夹角
    可求 Query 向量与文档特征向量的夹角
    sum(weight(i, j) * query(j)) / sqrt(sum(weight(i, j) * query(j)))
    长文档维度高，不公平
    特征权重计算
    Tf * IDF 框架
    词频因子 (Tf)
    抑制过大：W(Tf) = 1 + log(Tf)
    增强型规范化 Tf
    W(Tf) = a + (1 - a) * Tf / max(Tf)
    a ＝ 0.4 效果好，可抑制长文档
    逆文档频率因子 (IDF)
    特征词在整个文档集合中的区分度，含有信息量的多少
    IDF(k) = log(N / n(k))
    N = # of doc
    n = doc frequency
    Tf * IDF
    weight = Tf * IDF
    概率检索模型

    将文档划分为相关文档子集和不相关文档子集，如果属于一个子集的概率大就属于它

    Unary code: 对于整数 x，用 x-1 个 1 加一个表示结束的 0，共 x 个二进制数
    Binary code: n 位 bits 表示 2^n 内的数
    Elias Gamma: x = 2^e + d。 把 e+1 用 unary code，d 用 e 位 binary code
    Elias Delta: 继续把 e 也用 Elias Gamma 压缩
    Golomb & Rice: f1 = (x-1) / b, f2 = (x-1) mod b。然后对 f1+1 用 unary code，f2 用 binary code。Golomb 和 Rice 用不同的策略选择 b。局部 b 和全局 b。尚不明白为什么 x-1，继续压缩？范围从 0 开始？
