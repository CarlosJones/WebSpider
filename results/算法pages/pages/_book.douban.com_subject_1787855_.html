Programming from the Ground Up is an introduction to programming using assembly language on the Linux platform for x86 machines. It is a great book for novices who are just learning to program as well as for intermediates who have never learned or mastered assembly language programming. It covers many topics skipped by other similar books, including memory management, debugging, interfacing with C, dynamic libraries, and even a bit on GUI programming.

    Source code is the human-readable form of a program. In order to transform it into a program that a computer can run, we need to assemble and link it. The first step is to assemble it. Assembling is the process that transforms what you typed into instructions for the machine. The machine itself only reads sets of numbers, but humans prefer words. An assembly language is a more human-readable form of the instructions a computer understands. Assemble transforms the human-readable file into a machine-readable one.To assemble the program type in the command as exit.s -o exit.o as is the command which runs the assembler, exit.sis the source file, and-o exit.otells the assemble to put it’s output in the fileexit.o. exit.ois an object file. An object file is code that is in the machine’s language, but has not been completely put together. In most large programs, you will have several source files, and you will convert each one into an object file. Thelinker is the program that is responsible for putting the object files together and adding information to it so that the kernel knows how to load and run it. In our case, we only have one object file, so the linker is only adding the information to enable it to run. To link the file, enter the command ld exit.o -o exit ld is the command to run the linker, exit.ois the object file we want to link, and -o exit instructs the linker to output the new program into a file called exit. If any of these commands reported errors, you have either mistyped your program or the command. After correcting the program, you have to re-run all the commands. You must always re-assemble and re-link programs after you modify the source file for the changes to occur in the program.
    源代码经过编译(assemble)之后生成目标文件（.o 文件）， 目标文件中是机器语言，机器语言还需要经过链接(link) 之后将所需的目标文件组合并且加上内核加载运行该机器语言的相关必要信息之后才可以生成系统可执行文件。
