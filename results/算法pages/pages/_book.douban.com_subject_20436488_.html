内容简介 ······

深入剖析现代编译器运用的算法和技术

强调代码优化和代码生成

体现编译原理教学的最新理念

本书旨在介绍编译器构造法中的艺术和科学。书中深入分析现代编译器后端所用的算法和技术，重点讨论代码优化和代码生成，详细介绍了用几个编程语言编写的示例等。

Keith D. Cooper　莱斯大学计算机科学系计算工程专业Doerr特聘教授，曾任该系系主任。Cooper博士的研究课题涵盖过程间数据流分析、标量指令优化、寄存器分配以及指令调度等方面。

Linda Torczon　莱斯大学计算机科学系高级研究员。Torczon的研究内容主要包括代码生成、过程间数据流分析和优化、编程环境。

    402页，灰底之上倒数第三段末尾。
    当且仅当以下条件成立。
    随后是两个条件。原文这里的表述有歧义，很容易让人以为随后的两个条件任一成立即可，实际上它们需要同时成立。本章注释中提到该部分（10.2节）主要改编自Efficiently Computing Static Single Assignment Form and the Control Dependence Graph ，经检索取材于该论文（电子版 http://www.cs.utexas.edu/~pingali/CS380C/2010/papers/ssaCytron.pdf）的6节到7.1节，在这篇论文中对这两个条件的表为“A CFG node Y is control dependent on a CFG node X if both of the following hold:”，十分清晰。我以为要么直接将原文修订为"当且仅当以下条件同时成立"，要么添加译者注。
    接下来的403页问题更大：直接阅读原文将根本无法看懂“图10-1 消除无用代码”的算法描述！原文缺少对算法中使用的部分表达的说明，想要看懂还得参考上面的论文。
    406页，第三段第一句。
    与向Clean添加一个处理空循环的变换相比，这种协作更简单且更有效。这种变化可以识别...（This cooperation is simpler and more effective than adding a transformation to Clean that handles empty loops. Such a transformation might recognize...）
    第一句出现“这种写作“，第二句主语为”这种变换”指代向“Clean添加一个处理空循环的变换”。译文由于直译而没有考虑翻译为中文后，语序调整导致随后的句子的主语未调整，导致读起来异常拗口。
    409页。
    非译文问题，怀疑原文有错误，根据图10-4计算出来的DEExpr，UEExpr，ExprKill与书里提供的完全不同（简直是离谱地难以置信）。比如r1明显是定义的变量，属于VarKILL，为什么书里说它属于DEExpr或UEExpr？
    414页，倒数第三行。
    10.4节阐述的运算符强度削减
    实际应该是10.7节
    429页图10-11（以及431页图10-12）里第5行指令最后应为rs1。
    431页，倒数第二段。
    直观看来，归纳变量是值在循环中形成一个算数级数( arithmetic progression)的变量
    “算术级数”过于晦涩，建议改成广大人民群众更喜闻乐见的“等差数列”。
    归纳变量是SSA图的一个强连通分量
    这一句十分令人费解（非译文问题），归纳变量显然是一个变量，而强连通分量是有向图（一种数据结构）的极大连通子图。这两者完全属于不同的概念，能画等号？我揣测作者本意是指：更新归纳变量的多个节点会构成强连通分量。
    433页，图10-13 运算符强度削减算法。
    DFS方法的第五条语句for each operand o of n。怀疑这个表述有误，n是SSA图的一个节点，每个节点对应一条静态单赋值语句。既然是图，显然不能对n的每个operand编译，应该是 for each successor o of n，应遍历n的每个后继节点。
    438页，第二段。
    尽管受到搜索空间巨大的限制
    该句有歧义，“巨大”是修饰前面的空间，还是修饰后面的限制？参考原文“Despite the huge size of the search spaces”，可改为：“尽管搜索空间巨大”。

    361页，倒数第二段

    由于中文表达的习惯于英文不同，译文对原文的顺序进行调整，但译文对首次出现变量的说明未做调整，导致译文读起来十分别扭。具体来说有两点：首次出现的p前应添加“位置”、e前添加“表达式”，随后出现的则不必加；constituent subexpressions翻译做“成分-子表达式”不妥。

    更正式地说，节点n中的定义，仅在满足下述两个条件的汇合点才需要插入相应的函数∮

    接下来的403页问题更大：直接阅读原文将根本无法看懂“图10-1 消除无用代码”的算法描述！原文缺少对算法中使用的部分表达的说明，想要看懂还得参考上面的论文。

    第一句出现“这种写作“，第二句主语为”这种变换”指代向“Clean添加一个处理空循环的变换”。译文由于直译而没有考虑翻译为中文后，语序调整导致随后的句子的主语未调整，导致读起来异常拗口。
