    一个简单的String类的实现
    面试时，面试官可能要面试者写出一个简单的String类。下面一个简单的类实现开始我们今天的话题。
    #ifndef MY_STRING_H
    #define MY_STRING_H
    class String
    { private:
    char *str;
    int len;
    public:
    String();
    String(const String &s);
    String(const char* s);
    ~String();
    String & operator=(const String &s);
    String & operator=(const char* s);
    const char &operator[](int i)const;
    int Strlen();
    };
    #endif
    各个类成员函数的实现
    #include<cstring>
    #include "String.hpp"
    #include<iostream>
    using namespace std;
    //Construct String from c string
    String::String(const char *s)
    { len =strlen(s);
    str = new char[len+1];
    strcpy(str,s);
    } //copy constructor
    String::String(const String &s)
    { len = s.len;
    str = new char[len+1];
    strcpy(str,s.str);
    }
    //default constructor
    String::String()
    { zlen = 4;
    str = new char[1];
    str[0] = '\0';
    }
    String::~String()

    } //赋值操作符
    String& String::operator=(const String &s)
    { if(this==&s)
    return *this;
    delete [] str;
    len = strlen(s.str);
    str = new char[len+1];
    strcpy(str,s.str);

    { return len;
    } 别看这个类比较简单，但是里面涉及的知识点非常多。现在总结一下
    编译器给类都添加了什么东西
    深拷贝与浅拷贝怎么理解
    有关返回对象的说明
    编译器给类都添加了什么东西
    就算是你写一个空类，实际上在内部，编译器是添加了一些东西的，有一个是默认的构造函数，有一个是 copy assignment,有一个是copy constructor,在要明确一点，编译器默认的这些东西只有在程序中显示的调用时，才会出现。比如:
    String temp;//这样调用的是默认的构造函数,编译器自动生成默认的构造函数
    String temp(temp1)//这里的temp1也是一个String类，我们用这个类来初始化String类的时候，调用的是Copy costructor
    temp1 = temp;//而这样调用的是copy assignment
    默认的构造函数
    在没有提供任何的构造函数时，C++默认是提供了一个构造函数的，这个函数默认是不接受任何参数，也不干任何事情.
    Base example;//这样一个对象是没有被初始化的
    当然带参数的构造函数也可以是默认的构造函数，只要这些参数有默认的值
    赋值构造函数
    主要作用:
    主要将一个对象赋值到一个新创建的对象中，也就是说这个是用于初始化一个对象时，才会用到。类的赋值构造函数的原型通常是:class_name(const class_name &)
    何时调用赋值构造函数
    简单的例子
    String temp(tmp);
    String temp = tmp;
    String temp = String(tmp);
    String *temp = new String(tmp);
    总的说来，就是用一个现有的对象初始化新建的对象。我们应该注意的是，对于一些临时对象的建立，也需要调用默认构造函数。比较常见的就是在函数进行传值调用的时候，以及函数返回对象的时候，就会调用默认的赋值构造函数.
    我们要注意的是默认的构造函数是将非静态成员浅拷贝到新建的对象中。对于这一点我们要注意。有些情况下，浅拷贝是会带来一些问题的。在我们这个String类中，如果只是对str成员进行浅拷贝，那么就会出现两个对象的str成员同时指向一块相同的内存。这样，当其中一个对象被析构，那么这片内存将不存在。这样就引起了问题.所以，在写这个类的时候，我们要注意要进行深拷贝。面对这样的一个问题，我们就应该显示的声明copy构造函数。
    赋值操作符
    默认的行为跟赋值构造函数类似。也就是对类成员进行浅拷贝。
    技巧
    有的时候，我们可能不需要这些编译器默认的函数，对于出现显示调用时，我们希望编译器能够报错，提醒我们。我们可以显示的将这些函数放在private 中，并且我们也不去实现这些函数。
    深拷贝 vs 浅拷贝
    所谓的浅拷贝，就是讲一个类的成员赋值到另一个类的成员。在这个例子中就是将len，str的值赋值给另一个类。而深拷贝则不同，对于这个例子而言，我们将字符重新copy 到 另一块内存中.做到了字符串的真正拷贝！而不是简单的指针的值的拷贝。
    有关返回对象的说明
    返回指向对象的引用
    使用引用的目的是为了提高效率，要注意的是返回对象，要调用copy构造函数，而返回引用则不会。
    返回对象
    对于返回的对象是局部对象的变量，我们要注意的是不能给这种对象返回引用，应为当程序的控制权到调用的函数时，这个对象已经被销毁了。

    编译器给类都添加了什么东西
    深拷贝与浅拷贝怎么理解
    有关返回对象的说明
    编译器给类都添加了什么东西
    就算是你写一个空类，实际上在内部，编译器是添加了一些东西的，有一个是默认的构造函数，有一个是 copy assignment,有一个是copy constructor,在要明确一点，编译器默认的这些东西只有在程序中显示的调用时，才会出现。比如:
    String temp;//这样调用的是默认的构造函数,编译器自动生成默认的构造函数
    String temp(temp1)//这里的temp1也是一个String类，我们用这个类来初始化String类的时候，调用的是Copy costructor
    temp1 = temp;//而这样调用的是copy assignment
    默认的构造函数
    在没有提供任何的构造函数时，C++默认是提供了一个构造函数的，这个函数默认是不接受任何参数，也不干任何事情.
    Base example;//这样一个对象是没有被初始化的
    当然带参数的构造函数也可以是默认的构造函数，只要这些参数有默认的值
    赋值构造函数
    主要作用:
    主要将一个对象赋值到一个新创建的对象中，也就是说这个是用于初始化一个对象时，才会用到。类的赋值构造函数的原型通常是:class_name(const class_name &)
    何时调用赋值构造函数
    简单的例子
    String temp(tmp);
    String temp = tmp;
    String temp = String(tmp);
    String *temp = new String(tmp);
    总的说来，就是用一个现有的对象初始化新建的对象。我们应该注意的是，对于一些临时对象的建立，也需要调用默认构造函数。比较常见的就是在函数进行传值调用的时候，以及函数返回对象的时候，就会调用默认的赋值构造函数.
    我们要注意的是默认的构造函数是将非静态成员浅拷贝到新建的对象中。对于这一点我们要注意。有些情况下，浅拷贝是会带来一些问题的。在我们这个String类中，如果只是对str成员进行浅拷贝，那么就会出现两个对象的str成员同时指向一块相同的内存。这样，当其中一个对象被析构，那么这片内存将不存在。这样就引起了问题.所以，在写这个类的时候，我们要注意要进行深拷贝。面对这样的一个问题，我们就应该显示的声明copy构造函数。
    赋值操作符
    默认的行为跟赋值构造函数类似。也就是对类成员进行浅拷贝。
    技巧
    有的时候，我们可能不需要这些编译器默认的函数，对于出现显示调用时，我们希望编译器能够报错，提醒我们。我们可以显示的将这些函数放在private 中，并且我们也不去实现这些函数。
    深拷贝 vs 浅拷贝
    所谓的浅拷贝，就是讲一个类的成员赋值到另一个类的成员。在这个例子中就是将len，str的值赋值给另一个类。而深拷贝则不同，对于这个例子而言，我们将字符重新copy 到 另一块内存中.做到了字符串的真正拷贝！而不是简单的指针的值的拷贝。
    有关返回对象的说明
    返回指向对象的引用
    使用引用的目的是为了提高效率，要注意的是返回对象，要调用copy构造函数，而返回引用则不会。
    返回对象
    对于返回的对象是局部对象的变量，我们要注意的是不能给这种对象返回引用，应为当程序的控制权到调用的函数时，这个对象已经被销毁了。
