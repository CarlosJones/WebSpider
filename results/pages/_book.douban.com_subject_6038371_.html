JavaScript是Web开发中最重要的一门语言，它强大而优美。无论是桌面开发，还是移动应用。JavaScript都是必须掌握的技术。W3C的DOM标准是开发Web应用的基石。已经得到所有现代浏览器的支持，这使得跨平台Web开发成了一件轻松惬意的事。

本书是超级畅销书的升级版，由倡导Web标准的领军人物执笔，揭示了前端开发的真谛，是学习JavaScript和DOM开发的必读之作。

本 书在简洁明快地讲述JavaScript和DOM的基本知识之后，通过几个实例演示了专业水准的网页开发技术，透彻阐述了平稳退化等一批至关重要的 JavaScript编程原则和最佳实践，并全面探讨了HTML5以及jQuery等JavaScript库。读者将看到JavaScript、 HTML5和CSS如何协作来创建易用的、与标准兼容的Web设计，掌握使用JavaScript和DOM通过客户端动态效果和用户控制的动画来加强 Web页面的必备技术；同时，还将对如何利用库提高开发效率有全面深入的理解。

作者简介 ······

Jeremy Keith 国际知名的Web设计师，Web标准项目组成员，DOM Scripting任务组负责人之一，任职于Web咨询公司Clearleft。除本书外，还著有HTML5 for Web Designers、Bulletproof Ajax。可通过其个人网站adactio.com与他联系。

Jeffrey Sambells 资深平面设计师，Web开发人员，创办了软件公司We-Create并担任研发总监。除本书外，还著有《JavaScript高级程序设计》和 Beginning Google Maps Applications with PHP and Ajax等多部畅销书。

    第四章：案例研究：javascript图片库
    设计showPic函数
    通过改变"占位符"图片的src属性来替换参数图片，代码如下：
    function showPic(whichpic){
    var source = whichpic.getAttribut("href"); //把要显示的图片路径存入source变量
    var placeholder = document.getElementById("placeholder");//获取占位符图片对象，为了不反复敲getElementById而把对象赋给placeholder
    placeholder.setAttribute("src",source);//把source中的路径存进占位符图片的src属性中
    }
    非DOM方法也可实现，但是DOM的可移植性更好；
    javascript代码包含在一对引号之间，我们可以把任意数量的javascript语句放在这对引号之间，只要把各条语句用分号隔开即可：onclick = "showPci(this);return false;"
    childNodes属性
    可以用来获取任何一个元素的所有子元素，它是一个包含这个元素所有子元素的数组
    childeNodes属性返回的数组包含所有类型的节点，而不仅仅是元素节点。文档里几乎每一样东西都是一个节点，甚至连空格和换行符都会被解释为节点。
    nodeType属性
    每一个节点都有nodeType属性，这个属性可以让我们知道我们在于哪一种节点打交道，但是nodeType的值是数字而不是英文字符串；
    元素节点的nodeType值是1，属性节点的nodeType值是2，文本节点的nodeType值是3.
    nodeValue
    如果想改变一个文本节点的值，那就使用nodeValue属性
    注意一个细节：你想得到的是它第一个子节点的nodeValue属性值，直接调用nodeValue得到的是它自己的nodeValue值，正确的调用方法如下：alert(decription.childNode[0].nodeValue)
    firstChild和lastChild属性
    分别为第一个子节点喝最后一个子节点，node.firstNode等价于node.childNodes[0],node.lastNode等价于node.childNodes[node.childNodes.length-1]
    为了提高性能，要尽量减少访问DOM和尽量减少标记
    合并放置脚本
    推荐把多个JS脚本合并到一个JS脚本文件中，这样就可一减少加载页面时发送的请求数量。
    脚本在标记中的位置对页面的初次加载时间也有很大影响，传统上我们把脚本放在head区域，这种方法有一个问题，一般来说，根据HTTP规范，浏览器每次从同一个域名中最多只能同时下载两个文件。而在下载脚本期间，浏览器不会下载其他任何文件即使是来自不同域名的文件也不会下载，所有其他文件都要等脚本文件加载完毕后才能下载。所以，把所有script标签都放到文档的末尾，在/body标记之前，就可以让页面变得更快。即使这样，在加载脚本时，window对象的load事件依然可以执行对文档的各种操作。
    压缩代码
    为了精简代码，可以去掉空格、换行符等，多数情况下，你应该有两个版本，一个是工作副本，可以修改代码并且添加注释，另个是精简副本，用于放置在站点上。通常，为了与非精简版本区分开，最好在精简副本的文件名中加上min字样。
    代码压缩工具:JSMin;雅虎的YUI Compressor;谷歌的Closure Compiler。
    第五章：最佳实践（平稳退化、分离、向后兼容、性能考虑）
    window.open(url,name,features)
    javascript使用window对象的open()方法来创建新的浏览器窗口
    第一个参数是新窗口打开的网页的URL地址；第二个参数是新窗口的名字；最后一个参数是一个以逗号分隔的字符串，包含新窗口的各种属性
    例：
    function popUp(winURL){
    window.open(winURL,"popup","width=320,height=480");
    }
    "javascript:"伪协议
    在HTML文档里通过"javascript:"伪协议调用javascript代码的做法非常不好。
    平稳退化是为了照顾那些不支持javascript的用户，以及搜索引擎的爬虫程序。
    把JS代码存入外部文件时，HTML在加载头时，就将加载JS代码，此时可能有些DOM还未加载完毕，为了避免这种情况，通常将代码封装到一个函数里，在DOM加载完毕时加载这个函数，如下：
    window.onload = initAll;
    function initAll(){

    }
    对象检测
    使用如下语句来检测浏览器对javascript的支持程度：if(!getElementById || !getElementByTagName) return false;//对浏览器说：“如果你不理解getElementById和getElementByTagName，你就不能执行继续下面的任务了。”
    第六章：案例研究：图片库改进版
    共享onload事件
    前面提到将要执行的函数封装好后赋给window.onload，现在假设你有两个函数firstFunction和secondFunction，你希望他们两个都在页面加载时得到执行。如果把他们都绑定到onload事件上，他们当中将只有最后那个才会被实际执行。有个方法可以避过这一难题：可以创建一个匿名函数，然后把那个匿名函数绑定到onload事件上，如下：
    window.onload = function(){
    firstFunction();
    secondFunction();
    }
    还有另一个弹性最佳的解决方案，addLoadEvent函数，他只有一个参数：打算在页面加载完毕时执行的函数的名字。
    function addLoadFunction(func){
    var oldonload = window.onload; //把现有的window.onload事件处理函数的值存入变量oldonload
    if(typeof window.onload != 'function'){ //如果这个处理函数还没有绑定任何函数，就像平时那样把新函数添加给它
    window.onload = func;
    }
    else{ //如果这个处理函数已经绑定了一些函数，就把这新函数追加到现有指令的末尾
    window.onload = function(){
    oldonload();
    func();
    }
    }
    }
    这样之后，只需要添加一下代码就行了：addLoadEvent(firstFunction);addLoadEvent(secondFunction);

    第三章DOM
    获取元素
    getElementById
    注意大小写；参数为id属性。
    getElimentByTagName
    与getElementById有相似之处，但是返回的是一个数组,具有length属性，可以通过for循环来验证。
    允许把一个通配符作为参数，如你想知道文档中共有多少元素节点，只需用如下语句:alert(document.getElementByTagName(*).length);
    可与getElementById结合适用，如下：var shopping = document.getElementById("purchases");var items = shopping.getElementByTagName("*")
    getElementByClassName
    HTML5新增方法，可通过class属性中的类名来访问元素。方法比较新，有些可能没有实现，使用需小心。
    同样返回数组。
    可以查找带有多个类名的元素，要指定多个类名，只要在字符串参数中用空格分隔类名即可,但是最好不要这么做。如下：alert(document.getElementByClassName("important sale").length)
    同样可与上两个组合使用。
    获取和设置属性
    getAttribute
    参数为属性名；
    不属于document对象，所以不能通过document对象调用，只能通过元素节点调用；
    setAttribute
    对属性值进行修改
    object.setAttribute(attribute,value)有两个参数，第一个参数为属性名，第二个参数为属性值；
    通过setAttribute对文档做出修改后，通过浏览器查看源代码时看到的仍是改变签的属性。这种表里不一的现象源自DOM的工作模式：先加载文档的静态内容，再动态刷新，动态刷新不影响静态内容。这正式DOM的真正威力：对页面内容进行刷新却不需要在浏览器里刷新页面。

    ## 第1章 JavaScript简史
    DOM（文档对象模型）：一套对文档的内容进行抽象和概念化的方法。
    ## 第2章 JavaScript语法
    驼峰格式是函数名、方法名和对象属性名命名的首选格式。
    作者个人比较喜欢用双引号来包住字符串。
    数组可以用关键字 `Array` 声明。
    可以用不着明确地表明在创建数组，只需用一对方括号把各个元素的初始值括起来：
    var beatles = [ "John", "Paul", "George", "Ringo" ];
    可以通过在填充数组时为每个新元素明确地给出下标来创建`关联数组`。
    var lennon = Object();
lennon.name = "John";
lennon.year = 1940;
lennon.living = false;
    等价于
    var lennon = { name:"John", year:1940, living:false };
    在 if 语句中总是使用花括号是个好习惯。
    `===` 进行严格比较，不仅比较值，而且会比较变量的类型。
    在命名变量时，作者用下划线分隔各个单词；在命名函数时，则用驼峰命名法。
    如果在某个函数中使用了 `var`，那个变量就将被视为一个局部变量；反之，如果没有使用 `var`，那个变量就将被视为一个全局变量。
    对象：用户定义对象、内建对象、宿主对象。
    ## 第3章 DOM
    BOM（浏览器对象模型）
    节点：元素节点、文本节点、属性节点。
    获取元素：
    * `getElementById`
    * `getElementsByTagName`
    * `getElementByClassName`
    获取和设置属性：
    * `getAttribute()`
    * `setAttribute()`
    `getElementsByClassName` 适用于老浏览器的实现：
    function getElementsByClassName(node, classname) {
    if (node.getElementsByClassName) {
        // 使用现有方法
        return node.getElementsByClassName(classname);
    } else {
        var results = new Array();
        var elems = node.getElementsByTagName('*');
        for (var i = 0; i < elems.length; i++) {
            if (elems[i].className.indexOf(classname) != -1) {
                results[results.length] = elems[i];
            }
        }
        return results;
    }
}

    *DOM 方法*：
    `createElement` 方法：语法 `document.createElement(nodeName)`
    `appendChild` 方法：语法 `parent.append(child)`
    `createTextNode` 方法：语法 `document.createTextNode(text)`
    `insertBefore` 方法：把一个新元素插入到一个现有元素的前面。语法 `parentElement.insertBefore(newElement, targetElement)`
    自定义的 `insertAfter` 函数：
    function insertAfter(newElement, targetElement) {
    var parent = targetElement.parentNode;
    if (parent.lastChild == targetElement) {
        parent.appendChild(newElement);
    } else{
        parent.insertBefore(newElement, targetElement.nextSibling);
    }
}
    目标元素的下一个兄弟元素即目标元素的 `nextSibling` 属性。
    XMLHttpRequest 对象的 `readyState` 值：

    * 4 表示完成

    第1章 JavaScript简史
    DOM是一套对文档的内容进行抽象和概念化的方法。 W3C推出的标准化DOM可以让任何一种程序设计语言对使用任何一种标记语言编写出来的任何一份文档进行操控。 W3C对DOM的定义：一个与系统平台和编程语言无关的接口，程序和脚本可以通过这个接口动态地访问和修改文档的内容、结构和样式。

    #语法
    最好的做法是把<script>标签（可以使代码或者JS文件）放到HTML文档的最后，</body>标签之前。这样能使浏览器更快地加载页面。
    不需要写<script>标签的type="text/javascript"属性，因为脚本默认就是JavaScript，没必要再指定。
    ##语句
    建议在每条语句的末尾都加上一个分号，这是一种良好的编程习惯。
    ##注释
    单行用//，多行用/*……*/
    ##变量
    JavaScript允许直接对变量赋值而无需事先声明，但提前声明变量是一种良好的编程习惯。
    JavaScript区分大小写。
    变量名格式，可以在适当位置插入下划线，第二种是驼峰格式。
    通常，驼峰格式是函数名、方法名和对象属性名命名的首选格式。
    ##数据类型
    JavaScript是一种弱类型语言，可以在任何阶段改变变量的数据类型。
    字符串不管选择用双引号还是单引号，最好在整个脚本中保持一致。
    从某种意义来讲，为计算机设计程序就是与布尔值打交道。
    布尔值false和字符串"false"是两码事。
