    不知道的修饰符部分
    1. %o、%x 和 %X 即8进制和16进制整数总是作为无符号数处理。
    2. printf(s) 与 printf("%s", s)两者含义并不相同。前者字符串中的任何%字符视为一个格式项标志。如果除%%之外任何格式码在字符串s中出现，而后面又没有对应的参数，将会带来麻烦。
    3. 对于整数格式项 %d、%o、%x 和 %u， 精度修饰符指定了打印数字的最少位数。如果待打印的数值并不需要这么多位数的数字来表示，就会在它的前面补上0. 
    printf("%.2d/%.2d/%.4d\n", 7, 14, 1789); 
/* OUTPUT: 07/14/1789 */
    4. 对于%s格式项，精度修饰符指定了将要从相应的字符串值中打印的字符数。如果该字符串中包含的字符数少于精度修饰符所指定的字符数，输出的字符数就会少于精度修饰符指定的数目。如果需要，我们可以通过域宽修饰符来加长输出。 
    char name[14];
/* ... */
printf("... %.14s ...", ..., name, ...);
    这样就保证了name总能被正确的打印输出。
    5. 空白字符作为标志字符时，它的含义是：如果某数是一个非负数，就在它的前面插入一个空白字符。（负数则会显示负号）
    6. 可变域宽与精度。可以使用*同时替换域宽修饰符和精度修饰符，后面的参数列表中需要依次出现代码域宽、精度的参数及要打印的值。例如 
    printf("%*.*s\n", 12, 5, str);
printf("%12.5s\n", str);
    两条语句等价。
    如果*用于替换域宽修饰符，而与其对应参数的值为负数，那么效果相当于把负号作为"-"标志字符来处理。
    7. 新增的格式码（不算新增啦）。%n 用于指出已经打印的字符数，这个数被存储在对应参数（整形指针）所指向的整数中。
    8. 在大多数场合，我们都可以用 %. 来替换 %0，效果也非常接近。（表示怀疑）

    这个我之前真的不知道，而且，一般不会选择对一个文件既读又写，而是会操作两个文件。
    许多系统中的标准输入/输出库都允许程序打开一个文件，同时进行写入和读出的操作：
    FILE *fp;
fp = fopen(file, "r+");
    上面的例子代码打开了文件名由变量file指定的文件，对于存取权限的设定表明程序希望对这个文件进行输入和输出操作。
    编程者也许认为，程序一旦执行上述操作完毕，就可以自由地交错进行读出和写入的操作。遗憾的是，事实总难遂人所愿，为了保持与过去不能同时进行读写操作的程序的向下兼容性，一个输入操作不能随后直接紧跟一个输出操作，反之亦然。如果要同时进行输入和输出操作，必须在其中插入fseek函数的调用。

    不知道的修饰符部分
    1. %o、%x 和 %X 即8进制和16进制整数总是作为无符号数处理。
    2. printf(s) 与 printf("%s", s)两者含义并不相同。前者字符串中的任何%字符视为一个格式项标志。如果除%%之外任何格式码在字符串s中出现，而后面又没有对应的参数，将会带来麻烦。
    3. 对于整数格式项 %d、%o、%x 和 %u， 精度修饰符指定了打印数字的最少位数。如果待打印的数值并不需要这么多位数的数字来表示，就会在它的前面补上0. 
    printf("%.2d/%.2d/%.4d\n", 7, 14, 1789); 
/* OUTPUT: 07/14/1789 */
    4. 对于%s格式项，精度修饰符指定了将要从相应的字符串值中打印的字符数。如果该字符串中包含的字符数少于精度修饰符所指定的字符数，输出的字符数就会少于精度修饰符指定的数目。如果需要，我们可以通过域宽修饰符来加长输出。 
    char name[14];
/* ... */
printf("... %.14s ...", ..., name, ...);
    这样就保证了name总能被正确的打印输出。
    5. 空白字符作为标志字符时，它的含义是：如果某数是一个非负数，就在它的前面插入一个空白字符。（负数则会显示负号）
    6. 可变域宽与精度。可以使用*同时替换域宽修饰符和精度修饰符，后面的参数列表中需要依次出现代码域宽、精度的参数及要打印的值。例如 
    printf("%*.*s\n", 12, 5, str);
printf("%12.5s\n", str);
    两条语句等价。
    如果*用于替换域宽修饰符，而与其对应参数的值为负数，那么效果相当于把负号作为"-"标志字符来处理。
    7. 新增的格式码（不算新增啦）。%n 用于指出已经打印的字符数，这个数被存储在对应参数（整形指针）所指向的整数中。
    8. 在大多数场合，我们都可以用 %. 来替换 %0，效果也非常接近。（表示怀疑）
